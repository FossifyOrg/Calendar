# Contributing
Thank you so much for considering contributing to Fossify Calendar! We would love to have you contribute to our efforts as open-source contributions allow us to continue providing the most optimal user experience. Here is a list of starter tips to keep in mind when contributing:

1. When submitting your contributions, be sure to create a new branch and raise a pull request.
2. Within the pull request, make sure you are descriptive about the changes you are making to ensure the highest likelihood of success.
3. The team will carefully review your pull request to ensure it meets our guidelines. If any changes are determined to be necessary, we will reach out to you with specific feedback and suggestions to help you refine your pull request.
4. When contributing to Fossify Calendar, make sure to follow the rules laid out by Fossify otherwise your pull request has a high likelihood of not being accepted. In order to find ore information about what changes are acceptable, please reference the Fossify <a href="https://github.com/FossifyOrg/General-Discussion#contribution-rules-for-developers">Contribution Rules</a>


If you have any more questions concerning how to contribute, please reference the <a href="https://github.com/FossifyOrg/General-Discussion">General Discussion</a> page for Fossify and check out <a href = "https://github.com/FossifyOrg/Calendar/blob/master/CONTRIBUTING.md">Contributing.md</a>

## Get Started Contributing!

Here are some beginner steps to get started working on Fossify Calendar.

1. Fork the Repository
	- Forking the repository allows you to have your own personal copy that does not interfere with the current repository. This is where you will make your initial changes.
2. Clone the Repository
	- Use the following command in your command line to clone the repository to your local machine. </br>
	```git clone https://github.com/FossifyOrg/Calendar.git```
3. Create a feature branch
	- This makes pull requests easier as all of your changes will be organized on a single brain.
4. Commit to this feature branch
5. When ready, push your changes and draft up a pull request!
